# Lab 14: Basic iptables Usage

## Objective

The objective of this lab is to learn how to view, create, and manage basic firewall rules using iptables. You will disable UFW, create a simple firewall policy that allows SSH, DNS, and ICMP traffic while blocking all other traffic, verify the rules, and make them persistent across reboots. 

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - iptables
  - systemctl
  - apt
  - iptables-persistent

---

## Important Note

Before starting, create a VM snapshot.

This lab modifies your firewall configuration. If you accidentally block your own access, you can restore the snapshot and start again. 

---

## Step 1: View Existing Firewall Rules

Check the current iptables configuration.

```bash
sudo iptables -L
```

### Expected Result

Ubuntu normally has very few or no active iptables rules because UFW manages firewall settings.

**Screenshot 1:** Existing iptables rules.

---

## Step 2: Disable UFW

Disable the Uncomplicated Firewall service.

```bash
sudo systemctl disable --now ufw
```

Verify:

```bash
sudo systemctl status ufw
```

### Expected Result

The UFW service is inactive.

**Screenshot 2:** UFW disabled.

---

## Step 3: Clear Existing Rules

Flush all existing rules.

```bash
sudo iptables -F
```

Delete any user-defined chains.

```bash
sudo iptables -X
```

### Expected Result

The firewall starts from a clean configuration.

**Screenshot 3:** Empty firewall configuration.

---

## Step 4: Set Default Policies

Drop all incoming traffic by default.

```bash
sudo iptables -P INPUT DROP
```

Allow all outgoing traffic.

```bash
sudo iptables -P OUTPUT ACCEPT
```

Allow forwarded traffic to be dropped.

```bash
sudo iptables -P FORWARD DROP
```

### Verification

```bash
sudo iptables -L
```

### Expected Result

The default policies display:

```text
INPUT   DROP
FORWARD DROP
OUTPUT  ACCEPT
```

**Screenshot 4:** Default firewall policies.

---

## Step 5: Allow Established Connections

Allow responses to connections that have already been established.

```bash
sudo iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
```

### Expected Result

Established sessions continue to function normally.

**Screenshot 5:** Established connection rule.

---

## Step 6: Allow SSH Access

Permit SSH traffic.

```bash
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```

### Why?

Without this rule, remote SSH access would be blocked.

### Verification

```bash
sudo iptables -L
```

**Screenshot 6:** SSH rule.

---

## Step 7: Allow DNS Traffic

Permit DNS queries.

```bash
sudo iptables -A INPUT -p udp --sport 53 -j ACCEPT
```

```bash
sudo iptables -A INPUT -p tcp --sport 53 -j ACCEPT
```

### Expected Result

DNS responses are allowed.

**Screenshot 7:** DNS rules.

---

## Step 8: Allow ICMP Traffic

Allow ICMP packets such as ping.

```bash
sudo iptables -A INPUT -p icmp -j ACCEPT
```

### Verification

```bash
ping 8.8.8.8
```

### Expected Result

Ping requests and replies function correctly.

**Screenshot 8:** ICMP rule.

---

## Step 9: View the Firewall Rules

Display the current firewall rules.

```bash
sudo iptables -L
```

### Expected Result

The firewall now contains rules for:

- Established connections
- SSH
- DNS
- ICMP

All other incoming traffic is denied.

**Screenshot 9:** Configured firewall rules.

---

## Step 10: Add the Loopback Rule

The lab notes that the loopback interface was forgotten initially. Add it at the top of the rule list.

```bash
sudo iptables -I INPUT 1 -i lo -j ACCEPT
```

### Why?

Many Linux services communicate through the loopback interface (`127.0.0.1`). Blocking it can cause unexpected application failures.

### Verification

```bash
sudo iptables -L
```

```bash
sudo iptables -L -v
```

### Expected Result

The loopback rule appears as the first INPUT rule.

The `-v` option displays packet and byte counters.

**Screenshot 10:** Loopback rule and verbose output.

---

## Step 11: Install iptables-persistent

Install the package that saves firewall rules across reboots.

```bash
sudo apt update
```

```bash
sudo apt install iptables-persistent
```

When prompted:

```text
Save current IPv4 rules?  Yes
Save current IPv6 rules?  Yes
```

### Expected Result

The current firewall rules are saved.

**Screenshot 11:** Installation of iptables-persistent.

---

## Step 12: Reboot and Verify

Reboot the VM.

```bash
sudo reboot
```

After logging back in:

```bash
sudo iptables -L
```

### Expected Result

All firewall rules remain active after reboot.

**Screenshot 12:** Firewall rules after reboot.

---

## Results

A basic firewall was successfully created using iptables. Rules were configured to allow SSH, DNS, ICMP, and loopback traffic while denying all other incoming connections. The configuration was then saved using iptables-persistent and verified after a system reboot.

---

## Conclusion

This lab demonstrated the creation and management of a basic Linux firewall using iptables. The exercise showed how to implement a default-deny security posture while selectively permitting essential network services. Persisting the rules ensures that the firewall configuration remains active across system reboots, providing continuous protection for the system.
