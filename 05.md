# Lab 5: Hands-on Lab for Setting Account and Password Expiry Data

## Objective

The objective of this lab is to learn how to configure account expiration dates and password aging policies in Linux. These security controls help administrators manage user accounts, enforce regular password changes, disable inactive accounts, and reduce the risk of unauthorized access due to outdated credentials.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - useradd
  - usermod
  - passwd
  - chage

---

## Step 1: Create a User Account with an Expiration Date

Create a user account named **samson** with an account expiration date of **30 June 2027**.

```bash
sudo useradd -m -d /home/samson -s /bin/bash -e 2027-06-30 samson
```

Display the account aging information:

```bash
sudo chage -l samson
```

### Expected Result

The account is created successfully and the account expiration date is displayed.

**Screenshot 1:** Creation of the Samson account and initial account aging information.

---

## Step 2: Modify the Account Expiration Date

Change Samson's account expiration date to **31 July 2027**.

```bash
sudo usermod -e 2027-07-31 samson
```

Verify the change:

```bash
sudo chage -l samson
```

### Expected Result

The updated account expiration date is displayed.

**Screenshot 2:** Updated account expiration information.

---

## Step 3: Assign a Password

Assign a password to Samson's account.

```bash
sudo passwd samson
```

### Expected Result

The password is successfully assigned.

**Screenshot 3:** Password assignment for Samson.

---

## Step 4: Force Password Change on First Login

Force Samson to change the password during the next login.

```bash
sudo passwd -e samson
```

Verify the configuration:

```bash
sudo chage -l samson
```

### Expected Result

The password is marked as expired and must be changed during the next login.

**Screenshot 4:** Password expiration settings.

---

## Step 5: Log In as Samson

Switch to Samson's account.

```bash
su - samson
```

### Expected Result

The system prompts Samson to change the password before continuing.

After changing the password, return to the original account.

```bash
exit
```

**Screenshot 5:** Forced password change during first login.

---

## Step 6: Configure Password Aging Policies

Configure the following password aging settings:

- Minimum password age: 5 days
- Maximum password age: 90 days
- Inactive period after expiration: 2 days
- Warning period before expiration: 5 days

```bash
sudo chage -m 5 -M 90 -I 2 -W 5 samson
```

Verify the configuration:

```bash
sudo chage -l samson
```

### Expected Result

The configured password aging settings are displayed.

**Screenshot 6:** Password aging policy configuration.

---

## Step 7: Verify Account and Password Policies

Verify the complete account aging configuration.

```bash
sudo chage -l samson
```

Verify the password status.

```bash
sudo passwd -S samson
```

### Expected Result

The account expiration date, password expiration policy, inactivity period, warning period, and password status are displayed correctly.

**Screenshot 7:** Verification of account and password aging settings.

---

## Results

A user account named Samson was successfully created with a predefined expiration date. The account expiration date was modified, a password was assigned, and the user was forced to change the password during the first login. Password aging policies were then configured to enforce minimum and maximum password ages, inactivity periods, and warning periods before expiration.

---

## Conclusion

This lab demonstrated how Linux administrators can manage account expiration and password aging using the `useradd`, `usermod`, `passwd`, and `chage` utilities. These mechanisms improve system security by ensuring that user accounts expire when no longer needed and that passwords are changed regularly according to organizational security policies.
