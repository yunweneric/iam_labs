# Lab 9: Searching for SUID and SGID Files

## Objective

The objective of this lab is to learn how to identify files with SUID (Set User ID) and SGID (Set Group ID) permissions on a Linux system. These special permissions allow programs to run with elevated privileges and are commonly used by system utilities. Administrators should regularly audit SUID and SGID files because improperly configured files can become security risks.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - find
  - chmod
  - diff
  - ls

---

## Step 1: Search for Existing SUID and SGID Files

Search the entire filesystem for files with either SUID or SGID permissions and save the results.

```bash
sudo find / -type f -perm /6000 -ls > suid_sgid_files.txt
```

### Verification

```bash
wc -l suid_sgid_files.txt
head suid_sgid_files.txt
```

### Expected Result

A text file containing all discovered SUID and SGID files is created.

**Screenshot 1:** Initial SUID/SGID file inventory.

---

## Step 2: Create a Test File Under Another User Account

Switch to another user account on the system. You may use the **samson** account created in Lab 5 or another test account.

```bash
su - samson
```

Create a dummy shell script:

```bash
touch some_shell_script.sh
```

Assign the SUID permission:

```bash
chmod 4755 some_shell_script.sh
```

Verify:

```bash
ls -l some_shell_script.sh
```

Exit the account:

```bash
exit
```

### Expected Result

The file is created and displays the SUID permission bit.

**Screenshot 2:** Creation of the SUID test file.

---

## Step 3: Search for SUID and SGID Files Again

Run the search command a second time and save the results to a different file.

```bash
sudo find / -type f -perm /6000 -ls > suid_sgid_files_2.txt
```

### Verification

```bash
wc -l suid_sgid_files_2.txt
```

### Expected Result

The second inventory file is created successfully.

**Screenshot 3:** Updated SUID/SGID file inventory.

---

## Step 4: Compare the Results

Compare the two inventories.

```bash
diff suid_sgid_files.txt suid_sgid_files_2.txt
```

### Expected Result

The output displays the newly created file that was added between the first and second scans.

**Screenshot 4:** Difference between the two inventories.

---

## Step 5: Verify the SUID Permission

Locate the newly created file.

```bash
find /home -name some_shell_script.sh
```

Verify its permissions.

```bash
ls -l /home/samson/some_shell_script.sh
```

### Expected Result

The permission string contains an **s** in the owner's execute field, indicating that the SUID bit is set.

**Screenshot 5:** Verification of SUID permissions.

---

## Results

The system was successfully scanned for SUID and SGID files. A test file was created with the SUID permission bit enabled, and a second scan confirmed its presence. Comparing the two scan results demonstrated how administrators can identify changes to privileged files on a Linux system.

---

## Conclusion

This lab demonstrated how to locate SUID and SGID files, create a file with SUID permissions, and compare system inventories to detect changes. Regular auditing of SUID and SGID files is an important security practice because these files execute with elevated privileges and may become targets for privilege escalation attacks.
