# Lab 18: Firewall and DMZ Rules

## Objective

The objective of this lab is to study three classic **iptables** firewall architectures — a simple perimeter firewall, a DMZ design, and a transparent Squid proxy gateway — then explain how each works and adapt the concepts to your own lab network.

---

## Environment

- Operating System: Ubuntu 22.04 or AlmaLinux 9 (gateway/firewall VM)
- Tools Used:
  - iptables
  - A text editor
  - Optional: additional VMs for LAN and DMZ segments
- Reference Materials:
  - [Simple firewall example](https://www.cs.montana.edu/courses/spring2004/409/topics/nat/firewall.html)
  - [DMZ firewall example](https://www.cs.montana.edu/courses/spring2004/409/topics/nat/dmz.html)
  - [Transparent Squid proxy script](https://www.cyberciti.biz/tips/linux-setup-transparent-proxy-squid-howto.html) (also available locally as `fw.proxy.txt`)

### Step 6: Adapt the DMZ Rules

Sketch a DMZ layout for your lab network. Example:


Write the DNAT rules you would use, substituting your own IPs:


### Step 9: Experiment on Your Network (Optional)

If you have multiple VMs, test a subset of the proxy rules. On a gateway VM with two interfaces:

```bash
sudo sysctl -w net.ipv4.ip_forward=1
```

Apply a single DNAT rule (adjust IPs and interfaces):

```bash
sudo iptables -t nat -A PREROUTING -i eth1 -p tcp --dport 80 \
  -j DNAT --to 192.168.1.1:3128
```

Verify with:

```bash
sudo iptables -t nat -L -v
```

## Results

Three firewall architectures were reviewed and explained: a simple perimeter firewall with default-deny policies, a DMZ design with DNAT for published services, and a transparent Squid proxy gateway. Each design was analyzed for its strengths, rule flow, and security trade-offs. The concepts were adapted to a lab network topology.

---

## Conclusion

This lab showed that firewall design is an architectural decision, not just a collection of rules. A **simple firewall** protects a single gateway, a **DMZ** isolates public-facing services from the internal LAN, and a **proxy gateway** combines network redirection with application-layer access control. Understanding these patterns prepares administrators to design layered defenses that match organizational requirements.
