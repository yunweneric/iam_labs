# Lab 16: ip6tables

## Objective

The objective of this lab is to configure an IPv6 firewall using **ip6tables** while leaving the existing IPv4 iptables configuration unchanged. You will add mangle `PREROUTING` rules to block invalid IPv6 packets, test them with Nmap Windows and XMAS scans, and observe which rules are triggered.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - ip6tables
  - nmap
  - ip6tables-save

---

## Prerequisites

Complete [Labs 14](14.md) and [15](15.md) on the same Ubuntu VM. The IPv4 firewall configuration must remain in place.

---

## Step 1: View Existing IPv6 Rules

Check the current IPv6 firewall configuration.

```bash
sudo ip6tables -L
```

### Expected Result

Few or no ip6tables rules are defined by default.

**Screenshot 1:** Initial ip6tables rules.

---

## Step 2: Add Mangle PREROUTING Rules for IPv6

Add the same invalid-packet blocking logic used in Lab 15, adapted for IPv6.

```bash
sudo ip6tables -t mangle -A PREROUTING -m conntrack --ctstate INVALID -j DROP
```

```bash
sudo ip6tables -t mangle -A PREROUTING -p tcp ! --syn -m conntrack --ctstate NEW -j DROP
```

### Verification

```bash
sudo ip6tables -t mangle -L -v
```

### Expected Result

Two PREROUTING rules appear in the mangle table.

**Screenshot 2:** IPv6 mangle rules.

---

## Step 3: Save the IPv6 Ruleset

Save the rules to your home directory and copy them to the persistent location.

```bash
sudo ip6tables-save > rules.v6
```

```bash
sudo cp rules.v6 /etc/iptables/rules.v6
```

### Expected Result

The IPv6 rules are saved for persistence across reboots.

**Screenshot 3:** Saved rules.v6 file.

---

## Step 4: Obtain the VM's IPv6 Address

Display network interface addresses.

```bash
ip a
```

Note the global or link-local IPv6 address of your VM (for example, `fe80::a00:27ff:fe9f:d923`).

### Expected Result

At least one IPv6 address is listed for your network interface.

**Screenshot 4:** IPv6 address from ip a.

---

## Step 5: Run an IPv6 Windows Scan

From the machine where Nmap is installed, run a Windows scan against the VM's IPv6 address. Replace the address with your own.

```bash
sudo nmap -6 -sW fe80::a00:27ff:fe9f:d923
```

On the Ubuntu VM, check which rule was triggered:

```bash
sudo ip6tables -t mangle -L -v
```

### Expected Result

One of the mangle PREROUTING rules shows non-zero packet and byte counters.

**Screenshot 5:** Windows scan counter on ip6tables.

---

## Step 6: Run an IPv6 XMAS Scan

From the scanning machine:

```bash
sudo nmap -6 -sX fe80::a00:27ff:fe9f:d923
```

On the Ubuntu VM:

```bash
sudo ip6tables -t mangle -L -v
```

### Expected Result

The other mangle PREROUTING rule shows increased counters, indicating it handled the XMAS scan traffic.

**Screenshot 6:** XMAS scan counter on ip6tables.

---

## Step 7: Restore the VM Snapshot

Shut down the virtual machine and restore the snapshot taken at the start of [Lab 14](14.md).

```bash
sudo shutdown -h now
```

### Expected Result

The VM returns to the Lab 14 state, clearing the firewall experiments from Labs 15 and 16.

**Screenshot 7:** VM restored from snapshot.

---

## Results

An IPv6 mangle firewall was configured independently of the existing IPv4 rules. Nmap Windows and XMAS scans against the VM's IPv6 address triggered the invalid-packet drop rules. Packet counters in `ip6tables -t mangle -L -v` confirmed which rule handled each scan type.

---

## Conclusion

This lab demonstrated that IPv4 and IPv6 firewalls are managed separately on Linux using **iptables** and **ip6tables**. The same invalid-packet filtering principles apply to both protocols. As IPv6 adoption grows, administrators must configure both stacks to maintain consistent network access control.
