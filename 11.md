# Lab 11: Creating a Shared Group Directory

## Objective

The objective of this lab is to create a shared directory for a group of users and implement Linux permissions, SGID, sticky bit, and Access Control Lists (ACLs) to control file access. Users will create files, assign permissions, and test access restrictions within the shared directory.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - groupadd
  - useradd
  - passwd
  - chmod
  - chown
  - setfacl
  - getfacl
  - ls

---

## Step 1: Create the Sales Group

Create a group named **sales**.

```bash
sudo groupadd sales
```

### Verification

```bash
getent group sales
```

### Expected Result

The sales group is created successfully.

**Screenshot 1:** Creation of the sales group.

## Step 2: Create User Accounts

Create the users **mimi**, **mrgray**, and **mommy**, adding them to the sales group.

```bash
sudo useradd -m -d /home/mimi -s /bin/bash -G sales mimi
```

```bash
sudo useradd -m -d /home/mrgray -s /bin/bash -G sales mrgray
```

```bash
sudo useradd -m -d /home/mommy -s /bin/bash -G sales mommy
```

### Command Explanation

Example:

```bash
sudo useradd -m -d /home/mimi -s /bin/bash -G sales mimi
```


| Option        | Description                                         |
| ------------- | --------------------------------------------------- |
| sudo          | Executes the command with administrator privileges. |
| useradd       | Creates a new user account.                         |
| -m            | Creates the user's home directory automatically.    |
| -d /home/mimi | Specifies the user's home directory.                |
| -s /bin/bash  | Sets Bash as the default login shell.               |
| -G sales      | Adds the user to the supplementary sales group.     |
| mimi          | Username being created.                             |


### Verification

Verify the users were created successfully.

```bash
id mimi
```

```bash
id mrgray
```

```bash
id mommy
```

### Expected Result

The output displays the UID, GID, and group memberships for each user account.

**Screenshot 2:** User account creation and group membership verification.

## Step 4: Create the Shared Directory

Create the shared directory.

```bash
sudo mkdir /sales
```

Assign ownership.

```bash
sudo chown nobody:sales /sales

Apply permissions.
```

```bash
sudo chmod 3770 /sales
```

### Permission Explanation

The permission value **3770** consists of:


| Value | Meaning                   |
| ----- | ------------------------- |
| 3     | Sticky Bit (1) + SGID (2) |
| 7     | Owner has rwx permissions |
| 7     | Group has rwx permissions |
| 0     | Others have no access     |


The SGID bit ensures that newly created files inherit the group ownership of the directory.

The Sticky Bit prevents users from deleting files owned by other users.

### Verification

```bash
ls -ld /sales
```

### Expected Result

The directory displays permissions similar to:

```text
drwxrws--T
```

**Screenshot 4:** Shared directory configuration.

### Expected Result

**Screenshot 5:** Mimi creates a file.

## Step 6: Configure an ACL on Mimi's File

Restrict file permissions.

```bash
chmod 600 mimi_file.txt
```

Grant Mr. Gray read access.

```bash
setfacl -m u:mrgray:r mimi_file.txt
```

Verify the ACL configuration.

```bash
getfacl mimi_file.txt
```

```bash
ls -l mimi_file.txt
```

Exit Mimi's account.

```bash
exit
```

### Expected Result

Mr. Gray receives read permission through the ACL while other users remain restricted.

**Screenshot 6:** ACL configuration on Mimi's file.

## Results

A shared directory was successfully created and assigned to the sales group. Multiple users were added to the group and granted access to the directory. ACLs were used to provide selective access to files, while SGID ensured consistent group ownership and the Sticky Bit protected files from unauthorized deletion.

## Conclusion

This lab demonstrated how Linux groups, ACLs, SGID, and Sticky Bit permissions can be used together to create a secure collaborative workspace. These mechanisms allow multiple users to share resources while maintaining fine-grained control over file access, modification, and deletion.
