# Lab 6: Hands-on Lab for Detecting Compromised Passwords

## Objective

The objective of this lab is to learn how to determine whether a password has been exposed in known data breaches. Using the Have I Been Pwned (HIBP) Pwned Passwords API, administrators can check passwords against a large database of compromised credentials without revealing the actual password being tested. This helps users avoid passwords that are already known to attackers.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - curl
  - Bash shell script
  - Have I Been Pwned (HIBP) Password API

---

## Background

When passwords are leaked during data breaches, they often become publicly available and are reused by attackers in credential stuffing and password spraying attacks.

The Pwned Passwords API allows users to check whether a password has appeared in previous breaches using a privacy-preserving technique called **k-Anonymity**, where only a portion of the password hash is sent to the API.

---

## Step 1: Query the Pwned Passwords API

Use curl to query the API using the hash prefix provided in the lab.
