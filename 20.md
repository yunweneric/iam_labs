# Lab 20: Basic UFW Usage

## Objective

The objective of this lab is to learn how to manage a host firewall using the **Uncomplicated Firewall (UFW)** on Ubuntu. You will enable UFW, open specific ports, inspect how UFW translates rules into the underlying firewall backend, and explore the configuration files in `/etc/ufw/`.

---

## Environment

- Operating System: Ubuntu 22.04 (or Ubuntu 20.04)
- Tools Used:
  - UFW (`ufw`)
  - iptables / ip6tables (Ubuntu 20.04)
  - nftables (Ubuntu 22.04)

---

## Important Note

Restore your Ubuntu VM to a clean snapshot before starting. This removes any iptables, ip6tables, or nftables rules from Labs 14–19.

---

## Step 1: Verify the Firewall Is Clean

After rebooting from a clean snapshot, confirm no manual firewall rules are present.

On **Ubuntu 20.04**:

```bash
sudo iptables -L
```

On **Ubuntu 22.04**:

```bash
sudo nft list ruleset
```

### Expected Result

No custom firewall rules are active (UFW may show default inactive rules).

**Screenshot 1:** Clean firewall state.

---

## Step 2: View UFW Status

```bash
sudo ufw status
```

### Expected Result

UFW reports `Status: inactive`.

**Screenshot 2:** UFW inactive status.

---

## Step 3: Allow SSH and Enable UFW

Open port 22/TCP before enabling the firewall to avoid locking yourself out.

```bash
sudo ufw allow 22/tcp
```

Enable UFW:

```bash
sudo ufw enable
```

When prompted, confirm with `y`.

Check status:

```bash
sudo ufw status
```

### Expected Result

UFW is active and port 22/tcp is listed as allowed.

**Screenshot 3:** UFW enabled with SSH allowed.

---

## Step 4: Inspect the Underlying Firewall Rules

On **Ubuntu 20.04**:

```bash
sudo iptables -L
```

```bash
sudo ip6tables -L
```

On **Ubuntu 22.04**:

```bash
sudo nft list ruleset
```

### Expected Result

UFW has generated backend rules that permit SSH and implement a default-deny policy for other incoming traffic.

**Screenshot 4:** Backend rules after enabling UFW.

---

## Step 5: Allow DNS Traffic

Open port 53 for both TCP and UDP.

```bash
sudo ufw allow 53
```

Verify:

```bash
sudo ufw status
```

### Expected Result

Port 53 is listed for both protocols.

**Screenshot 5:** UFW status with DNS allowed.

---

## Step 6: Inspect Updated Backend Rules

On **Ubuntu 20.04**:

```bash
sudo iptables -L
```

```bash
sudo ip6tables -L
```

On **Ubuntu 22.04**:

```bash
sudo nft list ruleset
```

### Expected Result

New rules for port 53 appear in the backend firewall.

**Screenshot 6:** Backend rules with DNS entries.

---

## Step 7: Explore UFW Configuration Files

Change to the UFW configuration directory.

```bash
cd /etc/ufw/
```

```bash
ls -l
```

Review the key files:

```bash
less ufw.conf
```

```bash
less user.rules
```

```bash
less before.rules
```

### Expected Result

You can identify where UFW stores its main settings, user-defined rules, and pre-processing rules.

**Screenshot 7:** Contents of /etc/ufw/.

---

## Step 8: Review the IPv6 Before Rules

Examine the IPv6 equivalent of the before-rules file.

```bash
less /etc/ufw/before6.rules
```

### Expected Result

You understand that UFW maintains separate rule files for IPv4 and IPv6.

**Screenshot 8:** before6.rules contents.

---

## Step 9: Reload UFW

```bash
sudo ufw reload
```

### Expected Result

UFW reloads all rules from the configuration files without error.

**Screenshot 9:** UFW reload.

---

## Step 10: Observe All Firewall Tables

On **Ubuntu 20.04**:

```bash
sudo iptables -L
```

```bash
sudo iptables -t mangle -L
```

```bash
sudo ip6tables -L
```

```bash
sudo ip6tables -t mangle -L
```

On **Ubuntu 22.04**:

```bash
sudo nft list ruleset
```

### Expected Result

The complete set of UFW-managed rules is visible across all relevant tables.

**Screenshot 10:** Full backend ruleset.

---

## Step 11: Final Status Check

```bash
sudo ufw status verbose
```

### Expected Result

UFW shows active status with logging level, default policies, and allowed ports.

**Screenshot 11:** UFW verbose status.

---

## Results

UFW was enabled on a clean Ubuntu VM with SSH and DNS ports opened. The relationship between high-level `ufw allow` commands and the underlying iptables or nftables rules was observed. Configuration files in `/etc/ufw/` were reviewed to understand how UFW manages IPv4 and IPv6 rule sets.

---

## Conclusion

This lab demonstrated that **UFW** provides a user-friendly interface for host firewall management on Ubuntu. While administrators can work directly with iptables or nftables for fine-grained control (as in Labs 14–19), UFW simplifies day-to-day operations by abstracting rule syntax into intuitive commands. Understanding both layers — the UFW front end and the backend rules it generates — is essential for effective Linux firewall administration.
