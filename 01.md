# Lab 1: Hands-on Lab for Assigning Limited sudo Privileges

## Objective

The objective of this lab is to learn how to create Linux user accounts and assign different levels of administrative privileges using the **sudo** mechanism. By configuring the **sudoers** file, users can be granted full administrative access or restricted access to specific commands only. This demonstrates the principle of **least privilege**, which states that users should only be given the permissions necessary to perform their tasks. 

---

## Environment

- Operating System: AlmaLinux 9
- Tool Used: `sudo`, `visudo`
- Users Created:
  - lionel
  - katelyn
  - maggie

---

# Step 1: Create User Accounts

Create the three user accounts:

```bash
sudo useradd lionel
sudo useradd katelyn
sudo useradd maggie
```

Assign passwords:

```bash
sudo passwd lionel
sudo passwd katelyn
sudo passwd maggie
```

### Expected Result

The three user accounts are successfully created and assigned passwords. 

📸 **Screenshot 1:** User creation and password assignment.

---

# Step 2: Open the sudoers File

Use the safe editor for modifying sudo privileges:

```bash
sudo visudo
```

Locate the STORAGE command alias and uncomment it. 

Add the following lines at the end of the file:

```text
lionel ALL=(ALL) ALL
katelyn ALL=(ALL) /usr/bin/systemctl status sshd
maggie ALL=(ALL) STORAGE
```

Save and exit.

### Explanation

- **Lionel** receives full sudo privileges.
- **Katelyn** can only view the status of the SSH service.
- **Maggie** can execute commands defined in the STORAGE alias.

📸 **Screenshot 2:** Modified sudoers file.

---

# Step 3: Verify Lionel's Privileges

Switch to Lionel's account:

```bash
su - lionel
```

Test administrative access:

```bash
sudo su -
exit

sudo systemctl status sshd

sudo fdisk -l
```

Exit the account:

```bash
exit
```

### Expected Result

All commands execute successfully because Lionel has unrestricted sudo access. 

📸 **Screenshot 3:** Successful execution of root-level commands by Lionel.
