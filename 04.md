# Lab 4: Hands-on Lab for Setting Password Complexity Criteria

## Objective

The objective of this lab is to learn how to enforce password complexity requirements on a Linux system using the Password Quality (pwquality) framework. By configuring password policies, administrators can require users to create stronger passwords that are more resistant to dictionary attacks, brute-force attacks, and password guessing.

---

## Environment

* Operating System: Ubuntu 22.04
* Tool Used:

  * PAM (Pluggable Authentication Modules)
  * libpam-pwquality
  * pwquality.conf

---

## Step 1: Install the Password Quality Package

Install the password quality package:

```bash
sudo apt update
sudo apt install libpam-pwquality -y
```

### Expected Result

The package installs successfully and the system becomes capable of enforcing password complexity policies.

**Screenshot 1:** Installation of libpam-pwquality.

---

## Step 2: Configure Minimum Password Length

Open the password quality configuration file:

```bash
sudo nano /etc/security/pwquality.conf
```

Locate the following line and modify it:

```text
minlen = 19
```

Save the file and exit the editor.

### Expected Result

The system now requires passwords to contain at least 19 characters.

**Screenshot 2:** Modified pwquality.conf file showing the minlen setting.

---

## Step 3: Create a Test User

Create a new user account:

```bash
sudo useradd goldie
sudo passwd goldie
```

### Expected Result

The user account is successfully created.

**Screenshot 3:** Creation of the Goldie account.

---

## Step 4: Test the Minimum Length Policy

Attempt to assign the following passwords to Goldie's account:

```text
turkeylips
TurkeyLips
Turkey93Lips
```

### Expected Result

The system should generate warnings or reject passwords that do not meet the configured minimum length requirement.

**Screenshot 4:** Password validation messages.

---

## Step 5: Configure Password Complexity Rules

Open the password quality configuration file again:

```bash
sudo nano /etc/security/pwquality.conf
```

Comment out the minimum length setting and configure the following rules:

```text
# minlen = 19

minclass = 3
maxclassrepeat = 5
```

Save the file and exit.

### Explanation

* `minclass = 3` requires at least three character classes (uppercase, lowercase, digits, or special characters).
* `maxclassrepeat = 5` limits excessive repetition of the same character class.

**Screenshot 5:** Updated pwquality.conf file.

---

## Step 6: Test Password Complexity Enforcement

Attempt to assign several passwords that violate the configured policy.

Examples:

```text
aaaaaaaaaa
password123
ABCDEFG123
1111111111
```

Then test passwords that satisfy the policy.

Examples:

```text
SecurePass123
Ubuntu2025!
Goldie@Lab2025
```

### Expected Result

Weak passwords should generate warnings or be rejected, while stronger passwords should be accepted.

**Screenshot 6:** Password validation results for weak passwords.

**Screenshot 7:** Successful acceptance of strong passwords.

---

## Results

The password quality policy was successfully configured and enforced. The system rejected or warned against weak passwords and encouraged the use of stronger passwords that met the configured complexity requirements.

---

## Conclusion

This lab demonstrated how Linux systems can enforce password complexity requirements using the pwquality framework. By configuring minimum length and character-class requirements, administrators can improve account security and reduce the likelihood of successful password-based attacks.
