# Lab 3: Hands-on Lab for Creating an Encrypted Home Directory with adduser

## Objective

The objective of this lab is to learn how to create a Linux user account with an encrypted home directory using the eCryptfs file system. Encrypting a home directory protects a user's personal files from unauthorized access by ensuring that data stored on disk remains unreadable without proper authentication. This provides an additional layer of security for sensitive information stored on a Linux system.

---

## Environment

* Operating System: Ubuntu 22.04
* Tools Used:

  * eCryptfs
  * adduser
  * ecryptfs-utils

---

## Step 1: Install eCryptfs Utilities

Update the package repository and install the required package:

```bash
sudo apt update
sudo apt install ecryptfs-utils -y
```

### Expected Result

The package installs successfully and the system is prepared for encrypted home directories.

**Screenshot 1:** Installation of ecryptfs-utils.

---

## Step 2: Create a User with an Encrypted Home Directory

Create a user account with an encrypted home directory:

```bash
sudo adduser --encrypt-home cleopatra
```

You will be prompted to:

* Enter a password
* Confirm the password
* Provide optional user information

### Expected Result

The user account is created and an encrypted home directory is configured automatically.

**Screenshot 2:** Creation of the encrypted user account.

---

## Step 3: Verify the Home Directory

Display the contents of the `/home` directory:

```bash
ls -l /home
```

### Expected Result

The new user account appears in the list of home directories.

**Screenshot 3:** Output of `ls -l /home`.

---

## Step 4: Log In as the New User

Switch to the encrypted user account:

```bash
su - cleopatra
```

### Expected Result

The encrypted home directory is automatically mounted and decrypted after successful authentication.

**Screenshot 4:** Successful login as Cleopatra.

---

## Step 5: Retrieve the Encryption Passphrase

Display the encryption passphrase used by eCryptfs:

```bash
ecryptfs-unwrap-passphrase
```

### Expected Result

The system displays the underlying encryption key associated with the encrypted home directory.

**Screenshot 5:** Output of `ecryptfs-unwrap-passphrase`.

---

## Step 6: Create a Test File

Create a sample file within the encrypted home directory:

```bash
echo "This is a secret message" > ~/secret.txt
```

Verify that the file exists:

```bash
ls
```

### Expected Result

The file is successfully created within the user's home directory.

**Screenshot 6:** Creation of the test file.

---

## Step 7: Verify Home Directory Encryption

To verify that the home directory is encrypted, inspect the encrypted home structure from an administrator account and then verify access from the encrypted user account.

From the administrator account:

```bash
sudo ls -la /home/cleopatra/
```

Switch to the encrypted user account:

```bash
su - cleopatra
```

Verify the current working directory:

```bash
pwd
```

List the contents of the home directory:

```bash
ls
```

### Expected Result

The encrypted storage structure should be visible from the administrator account, while the authenticated user should be able to access files normally within the home directory.

**Screenshot 7:** Verification of encrypted home directory structure.

**Screenshot 8:** Verification of file accessibility from Cleopatra's account.

---

## Results

The eCryptfs package was successfully installed, and a user account named Cleopatra was created with an encrypted home directory. The encrypted directory was automatically mounted upon login, and the encryption passphrase was retrieved successfully. Verification confirmed the presence of the encrypted storage structure and demonstrated that authenticated users could transparently access files stored within the encrypted home directory.

---

## Conclusion

This lab demonstrated how to create and manage an encrypted home directory using eCryptfs. The encrypted directory protected user data while allowing transparent access after authentication. The exercise highlighted the importance of protecting data at rest and demonstrated a practical method for securing sensitive user files on Linux systems.
