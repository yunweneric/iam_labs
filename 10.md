# Lab 10: Working with Extended File Attributes

## Objective

The objective of this lab is to learn how to view and manage extended file attributes in Linux. Specifically, the lab demonstrates the behavior of the append-only (`a`) and immutable (`i`) attributes and how they affect file modification, deletion, renaming, and linking operations.

---

## Environment

* Operating System: Ubuntu 22.04
* Tools Used:

  * nano (or another text editor)
  * lsattr
  * chattr
  * rm
  * mv
  * ln

---

## Step 1: Create a Test File

Using your preferred text editor, create a file named `perm_demo.txt` containing a line of text.

Example:

```bash
nano perm_demo.txt
```

Add a line of text and save the file.

### Expected Result

The file `perm_demo.txt` is successfully created.

**Screenshot 1:** Creation of perm_demo.txt.

---

## Step 2: View the Extended Attributes

Display the file's current attributes.

```bash
lsattr perm_demo.txt
```

### Expected Result

The current extended attributes of the file are displayed.

**Screenshot 2:** Initial file attributes.

---

## Step 3: Add the Append-Only Attribute

Set the append-only (`a`) attribute.

```bash
sudo chattr +a perm_demo.txt
```

Verify the change.

```bash
lsattr perm_demo.txt
```

### Expected Result

The attribute listing shows the append-only flag.

**Screenshot 3:** Append-only attribute enabled.

---

## Step 4: Attempt to Overwrite and Delete the File

Try the following commands:

```bash
echo "I want to overwrite this file." > perm_demo.txt
```

```bash
sudo echo "I want to overwrite this file." > perm_demo.txt
```

```bash
rm perm_demo.txt
```

```bash
sudo rm perm_demo.txt
```

### Expected Result

The overwrite and deletion attempts fail because the file is protected by the append-only attribute.

**Screenshot 4:** Failed overwrite and deletion attempts.

---

## Step 5: Append Data to the File

Append a line to the file.

```bash
echo "I want to append this line to the end of the file." >> perm_demo.txt
```

### Expected Result

The new text is successfully appended to the file.

**Screenshot 5:** Successful append operation.

---

## Step 6: Remove the Append-Only Attribute and Enable Immutable Mode

Remove the append-only attribute.

```bash
sudo chattr -a perm_demo.txt
```

Verify:

```bash
lsattr perm_demo.txt
```

Enable the immutable attribute.

```bash
sudo chattr +i perm_demo.txt
```

Verify:

```bash
lsattr perm_demo.txt
```

### Expected Result

The immutable flag is displayed in the file attributes.

**Screenshot 6:** Immutable attribute enabled.

---

## Step 7: Repeat the Overwrite and Delete Tests

Repeat the commands from Step 4.

```bash
echo "I want to overwrite this file." > perm_demo.txt
```

```bash
sudo echo "I want to overwrite this file." > perm_demo.txt
```

```bash
rm perm_demo.txt
```

```bash
sudo rm perm_demo.txt
```

### Expected Result

All modification and deletion attempts fail because the file is immutable.

**Screenshot 7:** Failed modification attempts on immutable file.

---

## Step 8: Attempt to Rename and Hard-Link the File

Try the following commands:

```bash
mv perm_demo.txt some_file.txt
```

```bash
sudo mv perm_demo.txt some_file.txt
```

```bash
ln ~/perm_demo.txt ~/some_file.txt
```

```bash
sudo ln ~/perm_demo.txt ~/some_file.txt
```

### Expected Result

The rename and hard-link operations fail due to the immutable attribute.

**Screenshot 8:** Rename and hard-link attempts.

---

## Step 9: Create a Symbolic Link

Attempt to create a symbolic link.

```bash
ln -s ~/perm_demo.txt ~/some_file.txt
```

### Expected Result

Observe and record the result of the symbolic link creation attempt.

**Screenshot 9:** Symbolic link creation attempt.

---

## Results

The append-only and immutable file attributes were successfully applied and tested. The append-only attribute allowed new data to be added while preventing overwriting and deletion. The immutable attribute provided stronger protection by preventing modifications, deletions, renaming, and hard-link creation.

---

## Conclusion

This lab demonstrated how Linux extended file attributes can be used to protect important files from accidental or unauthorized changes. The append-only attribute is useful for log files, while the immutable attribute provides a higher level of protection for critical system and configuration files.
