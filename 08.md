# Lab 8: Installing and Configuring FreeIPA and ADSys on AlmaLinux

## Objective

The objective of this lab is to install and configure FreeIPA and ADSys. FreeIPA provides centralized identity management, authentication, authorization, and policy services, while ADSys enables integration with centralized policy management environments.

---

## Environment

- Operating System: AlmaLinux 9
- Tools Used:
  - FreeIPA Server
  - ADSys
  - Kerberos
  - LDAP

---

# Part A: Installing and Configuring FreeIPA

## Step 1: Update the System

```bash
sudo dnf update -y
sudo reboot
```

### Expected Result

The system packages are updated successfully.

**Screenshot 1:** System update completed.

---

## Step 2: Set the Hostname

Configure the server hostname.

```bash
sudo hostnamectl set-hostname ipa.almalab.local
```

Verify:

```bash
hostnamectl
```

### Expected Result

The hostname is configured correctly.

**Screenshot 2:** Hostname configuration.

---

## Step 3: Configure the Hosts File

Edit:

```bash
sudo nano /etc/hosts
```

Add:

```text
127.0.0.1 localhost
192.168.1.100 ipa.almalab.local ipa
```

### Verify

```bash
ping ipa.almalab.local
```

### Expected Result

Hostname resolves correctly.

**Screenshot 3:** Hosts file configuration.

---

## Step 4: Install FreeIPA Server

Install FreeIPA packages.

```bash
sudo dnf install ipa-server ipa-server-dns -y
```

### Verify Installation

```bash
ipa-server-install --help
```

### Expected Result

The installation utility is available.

**Screenshot 4:** FreeIPA packages installed.

---

## Step 5: Configure FreeIPA Server

Run:

```bash
sudo ipa-server-install
```

Provide values similar to:

```text
Realm: ALMALAB.LOCAL
Domain: almalab.local
Server Hostname: ipa.almalab.local
```

### Expected Result

The installer configures:

- LDAP
- Kerberos
- DNS
- Certificate Authority
- Web Administration Interface

**Screenshot 5:** FreeIPA installation completed.

---

## Step 6: Verify FreeIPA

Obtain an administrative Kerberos ticket.

```bash
kinit admin
```

Verify:

```bash
klist
```

### Expected Result

A valid Kerberos ticket is displayed.

**Screenshot 6:** Kerberos authentication.

---

## Step 7: Create a Test User

Create a centralized user.

```bash
ipa user-add student1 \
--first=Student \
--last=One \
--password
```

Verify:

```bash
ipa user-find
```

### Expected Result

The user account appears in the FreeIPA directory.

**Screenshot 7:** User creation.

---

# Part B: Installing and Configuring ADSys

## Step 8: Install ADSys

Install ADSys.

```bash
sudo dnf install adsys -y
```

### Verify Installation

```bash
adsysctl version
```

### Expected Result

The ADSys utility is installed successfully.

**Screenshot 8:** ADSys installation.

---

## Step 9: Enable ADSys Service

Start and enable the service.

```bash
sudo systemctl enable --now adsys
```

Verify:

```bash
sudo systemctl status adsys
```

### Expected Result

The ADSys service is active and running.

**Screenshot 9:** ADSys service status.

---

## Step 10: Verify ADSys

Check ADSys functionality.

```bash
adsysctl policy debug
```

### Expected Result

Policy information is displayed successfully.

**Screenshot 10:** ADSys verification.

---

# Results

FreeIPA was successfully installed and configured on AlmaLinux. The server was able to provide centralized identity management services, including user authentication and Kerberos ticket management. ADSys was installed and configured successfully, enabling centralized policy management capabilities.
