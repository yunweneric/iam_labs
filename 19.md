# Lab 19: nftables on Ubuntu

## Objective

The objective of this lab is to configure a workstation firewall using **nftables** on Ubuntu. You will disable UFW, build a ruleset based on the workstation template, add `prerouting` rules to drop invalid packets early, test the configuration with Nmap scans, and verify dropped packets in the kernel log.

---

## Environment

- Operating System: Ubuntu 22.04
- Tools Used:
  - nftables (`nft`)
  - nmap
  - `/var/log/kern.log`

---

## Important Note

Restore your Ubuntu VM to a clean snapshot before starting. This clears any iptables or ip6tables rules from Labs 14–16.

---

## Step 3: Edit the Top Portion of nftables.conf

Open the configuration file.

```bash
sudo nano /etc/nftables.conf
```

Replace the top portion of the file with the following. This adds a **mangle** table with `prerouting` rules that drop invalid packets before connection tracking completes:

```nft
#!/usr/sbin/nft -f

flush ruleset

table inet mangle {
    chain prerouting {
        type filter hook prerouting priority -300;

        ct state invalid counter log prefix "Invalid Packets: " drop
        tcp flags & (fin|syn|rst|ack) != syn ct state new counter log prefix "Invalid Packets: " drop
    }
}
```

### Explanation

| Rule | Blocks |
|------|--------|
| `ct state invalid` | Packets that fail connection tracking validation |
| `tcp flags != syn` with `ct state new` | New TCP connections that do not start with SYN |

Both rules log dropped packets with the prefix `Invalid Packets:`.

**Screenshot 3:** Top portion of nftables.conf.

---

## Step 4: Edit the Second Portion — Filter Table and Input Chain

Continue editing `/etc/nftables.conf`. Add the filter table with an `input` chain:

```nft
table inet filter {
    set tcp_open_ports {
        type inet_service
        flags interval
        elements = { 22 }
    }

    chain input {
        type filter hook input priority 0; policy drop;

        ct state established,related accept
        ct state invalid drop

        iif lo accept

        ip protocol icmp icmp type {
            echo-reply, echo-request, time-exceeded,
            parameter-problem, destination-unreachable
        } accept

        ip6 nexthdr ipv6-icmp icmpv6 type {
            echo-reply, echo-request, time-exceeded,
            parameter-problem, destination-unreachable,
            packet-too-big
        } accept

        ip6 nexthdr ipv6-icmp icmpv6 type {
            nd-router-advert, nd-router-solicit,
            nd-neighbor-advert, nd-neighbor-solicit
        } ip6 hoplimit 255 accept

        tcp dport @tcp_open_ports ct state new accept
        udp dport 53 ct state new accept

        counter log prefix "[INPUT DROP]: " drop
    }
```

**Screenshot 4:** Input chain configuration.

---

## Step 5: Edit the Final Portion — Forward and Output Chains

Complete the file with forward and output chains:

```nft
    chain forward {
        type filter hook forward priority 0; policy drop;
    }

    chain output {
        type filter hook output priority 0; policy accept;
    }
}
```

Save and exit.

### Expected Result

The complete `/etc/nftables.conf` defines mangle prerouting, filter input, forward, and output chains.

**Screenshot 5:** Complete nftables.conf file.

## Step 6: Validate and Load the Ruleset

Check syntax without applying:

```bash
sudo nft -c -f /etc/nftables.conf
```

If validation passes, reload nftables:

```bash
sudo systemctl enable nftables
```

```bash
sudo systemctl reload nftables
```

### Verification

```bash
sudo nft list tables
```

```bash
sudo nft list table inet filter
```

```bash
sudo nft list ruleset
```

### Expected Result

Both `inet mangle` and `inet filter` tables are active.

**Screenshot 6:** Loaded nftables ruleset.

## Step 7: Run a Windows Scan

On the Ubuntu VM, check which prerouting rule was triggered:

## Results

A workstation nftables ruleset was built from the Ubuntu template with added mangle prerouting rules for early invalid-packet filtering. Nmap Windows and NULL scans triggered different prerouting rules, as confirmed by packet counters and kernel log entries.

---

## Conclusion

This lab demonstrated that **nftables** is the modern replacement for iptables on Ubuntu, offering a cleaner syntax and unified IPv4/IPv6 support. Placing invalid-packet drop rules in a `prerouting` chain at priority `-300` processes malicious traffic before it reaches the main filter chains, improving both security and performance.
